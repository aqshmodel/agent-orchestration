
# A.G.I.S. - AI Generative Intelligence System

A.G.I.S. (AI Generative Intelligence System) は、階層的なAI組織をシミュレートするインタラクティブなアプリケーションです。1人のプレジデントAIと60人の専門家エージェントが連携し、ユーザーからの複雑な要求に対して包括的な回答を生成します。

## 動作フロー (How it Works)

1.  **指示:** ユーザーがプレジデントにプロジェクトの目標や課題を指示します。
2.  **チーム編成と戦略立案:** プレジデントは指示を解釈し、Google検索で最新情報を収集。最適な専門家チームを選出し、統括管理者であるオーケストレーターに高レベルの戦略を伝達します。
3.  **タスク実行と組織学習:** オーケストレーターは戦略を具体的なタスクに分解し、専門家エージェントを並列または直列で呼び出します。プロジェクトの進行中、当初のチームでは専門知識が不足していると判断した場合、オーケストレーターは自律的に新たな専門家をチームに招集します（自己組織化）。 必要に応じて、エージェント間の相談（コンサルテーション）を仲介し、連携を促進します。さらに、オーケストレーターは、重要な成果物に対して他の専門家による相互レビューを指示し、アウトプットの品質と多角性を保証します。各エージェントは互いの成果をコンテキストとして参照しながら、それぞれの専門分野のタスクを実行します。**さらに、各エージェントが生み出した重要な知見（キーインサイト）は、リアルタイムで組織の『共有知識ベース』に蓄積されます。新たにタスクを開始するエージェントは、この知識ベースを最優先で参照することで、組織としての学習を加速させます。**
4.  **統合と報告:** 各エージェントの報告が完了すると、オーケストレーターはそれらを一つの報告書に統合し、プレジデントに提出します。
5.  **レビューと最終化:** プレジデントは報告書をレビューします。内容が不十分な場合は、オーケストレーターに追加の指示を与えてタスクを継続させます。内容が完璧であれば、プレジデント自らがユーザーのための詳細な最終レポートを作成し、ミッション完了となります。

## エージェント構成 (The Agents)

A.G.I.S.は、1つのリーダーシップチームと10の専門家チーム、合計62体のエージェントで構成されています。

### リーダーシップチーム (Leadership Team)
組織の意思決定とプロジェクト管理を担う中核チーム。

| 名前 (ID) | 役割 | 主な責任 |
| :--- | :--- | :--- |
| **プレジデント (president)** | 最高意思決定者 | ユーザーの指示を解釈し、最適なチームを編成。オーケストレーターに初期指令を与え、最終報告書をレビュー・完成させる。 |
| **プロジェクト・オーケストレーター (orchestrator)** | 統括管理者 | プレジデントの指令に基づき、タスクを分解し、専門エージェントを動員。プロジェクトの進行を管理し、人間との対話も担当する。 |

| A11 | トレンド・フォーキャスター | 社会や技術の中長期的トレンドを予測し、複数の未来シナリオを提示することで、長期戦略を支援する。 |
